Лабораторна робота №3

Тема: Реалізація стеганографії методом LSB у цифровому зображенні

1. Мета роботи

Метою лабораторної роботи є реалізація алгоритму приховування текстової інформації в цифровому зображенні методом LSB (Least Significant Bit), самостійна розробка функцій кодування та декодування повідомлення, а також проведення аналізу змін між оригінальним та модифікованим зображенням.

2. Постановка задачі

Необхідно:

Продемонструвати розуміння процесу стеганографії через покроковий алгоритм.

Реалізувати функції:

hide_message() — для приховування повідомлення;

extract_message() — для витягування повідомлення.

Показати роботу програми на власних персональних даних.

Провести аналіз змін зображення (розмір файлу, кількість змінених пікселів, числові метрики).

3. Теоретичні відомості

Метод LSB (Least Significant Bit) базується на заміні найменш значущого біта кожного кольорового каналу пікселя (R, G, B) на біти прихованого повідомлення.

Оскільки зміна LSB призводить до відхилення значення кольору максимум на ±1, візуальні зміни зображення практично непомітні для людського ока.

Ємність зображення визначається формулою:

Capacity = width × height × 3 (біт)

де:

width — ширина зображення,

height — висота,

3 — кількість кольорових каналів (RGB).

4. Опис алгоритму (Етап 1)
Приховування повідомлення:

Повідомлення кодується у формат UTF-8.

Обчислюється довжина повідомлення в байтах.

Довжина записується у перші 32 біти (заголовок).

Повідомлення перетворюється у бітовий потік.

Перевіряється достатність ємності зображення.

Біти послідовно записуються у LSB каналів R, G, B.

Результат зберігається у форматі PNG.

Витягування повідомлення:

Зчитуються LSB усіх пікселів.

Перші 32 біти визначають довжину повідомлення.

Відповідна кількість біт перетворюється назад у байти.

Байти декодуються у текст UTF-8.

5. Реалізація (Етап 2)

У роботі реалізовано такі основні функції:

hide_message(image_path, message, output_path) — приховує текст у зображенні.

extract_message(image_path) — витягує приховане повідомлення.

analyze_images(original_path, stego_path) — аналізує зміни між файлами.

Використані бібліотеки:

Pillow (PIL) — для роботи із зображеннями;

os — для визначення розміру файлів;

math — для обчислення PSNR;

argparse — для роботи з аргументами командного рядка.

Готові бібліотеки стеганографії не використовувалися. Логіка алгоритму реалізована самостійно.

6. Демонстрація роботи (Етап 3)

Під час виконання програма запитує персональні дані користувача:

ім’я / ПІБ,

дату народження,

навчальну групу.

Сформований текст приховується у зображенні та після цього витягується для перевірки коректності роботи алгоритму.

7. Аналіз змін зображення (Етап 4)

Програма здійснює порівняння оригінального та стего-зображення за такими параметрами:

розмір файлу (у байтах),

роздільна здатність,

кількість змінених пікселів,

відсоток змінених пікселів,

MSE (Mean Squared Error),

PSNR (Peak Signal-to-Noise Ratio).

Оскільки змінюється лише найменш значущий біт, максимальна зміна кожного каналу становить ±1.

8. Інструкція з запуску
Повна демонстрація:
python lab3_lsb.py
Використання власного зображення:
python lab3_lsb.py --in input.png --out stego.png
Витягування повідомлення:
python lab3_lsb.py --in stego.png --extract

Увага: для коректної роботи необхідно використовувати формат PNG (формат JPEG призводить до втрати даних через стиснення з втратами).

9. Висновок

У результаті виконання лабораторної роботи було реалізовано алгоритм стеганографії методом LSB без використання готових спеціалізованих бібліотек. Розроблено функції приховування та витягування повідомлення, підтверджено їхню коректність на персональних даних користувача та проведено аналіз змін зображення.

Отримані результати демонструють, що метод LSB забезпечує ефективне приховування інформації з мінімальними візуальними спотвореннями зображення.
